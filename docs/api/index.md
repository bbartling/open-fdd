# API Reference

The platform **REST API** is documented in one place: **[Appendix: API Reference](appendix/api_reference)**. It summarizes CRUD (sites, equipment, points), config, data model, download, analytics, BACnet proxy, faults, and jobs. Use the **React frontend** (http://localhost:5173) for day-to-day workflows.

**Interactive docs:** When the API is running, open **Swagger UI** at [http://localhost:8000/docs](http://localhost:8000/docs) or **ReDoc** at [http://localhost:8000/redoc](http://localhost:8000/redoc). Full OpenAPI 3.1 spec at [http://localhost:8000/openapi.json](http://localhost:8000/openapi.json). When `OFDD_API_KEY` is set, click **Authorize** in Swagger and paste the key from `stack/.env`.
