# Auth secrets and password bootstrap

Open-FDD stores edge credentials in `workspace/auth.env.local` (gitignored). Generate them with the Rust CLI — no Python required.

## Generate credentials

**Recommended (no Rust on host):**

```bash
./scripts/openfdd_auth_init.sh --show-secrets
```

Uses `openfdd-edge` from your local bridge image or GHCR to write bcrypt hashes to `workspace/auth.env.local`.

**With Rust toolchain installed:**

```bash
openfdd-edge auth generate --out workspace/auth.env.local --show-secrets
```

Or one-shot security bootstrap (auth + TLS certs):

```bash
openfdd-edge bootstrap security --show-secrets
```

**GHCR-only host (no source checkout):**

```bash
./scripts/openfdd_auth_init.sh --ghcr --force --show-secrets
```

Generated file includes bcrypt hashes only:

- `OFDD_AUTH_SECRET`
- `OFDD_OPERATOR_USER` / `OFDD_OPERATOR_PASSWORD_HASH`
- `OFDD_INTEGRATOR_USER` / `OFDD_INTEGRATOR_PASSWORD_HASH`
- `OFDD_AGENT_USER` / `OFDD_AGENT_PASSWORD_HASH`
- `OFDD_ADMIN_USER` / `OFDD_ADMIN_PASSWORD_HASH`
- `OPENFDD_AUTH_TTL_SEC=28800`

Plaintext passwords are printed once with `--show-secrets` and are never written to disk or logs.

## Rotate credentials

```bash
./scripts/openfdd_auth_init.sh --rotate --all --show-secrets
docker compose -f docker-compose.local.yml up -d --force-recreate openfdd-bridge
```

Or with `openfdd-edge` on PATH:

```bash
openfdd-edge auth rotate --role agent --show-secrets
openfdd-edge auth rotate --all --show-secrets
```

Rotation backs up the previous file to `auth.env.local.bak.<timestamp>`, preserves usernames unless you edit them manually, and rotates `OFDD_AUTH_SECRET` when using `--all` (invalidates existing JWTs).

## IT-managed passwords

Set hashes manually in `workspace/auth.env.local`:

```bash
openfdd-edge auth hash-password 'Your-Strong-Password-Here'
```

Paste the `$2…` output into `OFDD_*_PASSWORD_HASH`. Legacy plaintext `OFDD_*_PASSWORD` keys are still accepted for migration but are not generated by default.

## Roles

| Role | Typical use |
|------|-------------|
| operator | Read-only dashboard / exports |
| integrator | Configuration, imports, field-bus writes (with approval) |
| agent | Automation / agent API |
| admin | Full access including agent-only maintenance |

Agent credentials must remain separate from human operator accounts.

## Never commit secrets

- `workspace/auth.env.local` is in `.gitignore`
- Do not paste generated passwords into docs, tests, or issues
- Use `chmod 600 workspace/auth.env.local` on Linux/macOS
